# Repo Work Skill

## Goal
Help inspect, understand, modify, test, and document a software repository safely and efficiently.

## Workflow

### 1. Understand the task
Clarify:
- What should be changed or investigated
- Expected behavior
- Relevant files, feature area, issue, or failing test
- Constraints: no refactor, minimal diff, backward compatibility, etc.

### 2. Inspect the repo
Before editing:
- Read `README`, contribution docs, package/build files
- Identify stack, test framework, scripts, linting
- Search for related code, tests, configs, and similar patterns
- Prefer existing conventions over new abstractions
